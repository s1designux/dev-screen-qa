# 개발화면 검수 (dev-screen-qa)

피그마 디자인(정답)과 실제 개발화면을 대조해 **틀린 곳을 찾아내는 검수 도구**입니다.

## 지금 쓰는 것 → `plugin-value-qa/`  ⭐

**값 기반 검수 Figma 플러그인.** 사진(픽셀) 비교가 아니라 **실제 값**(색·크기·위치·글자)을 비교합니다.
- 피그마 정답 프레임의 값을 라이브로 읽고, 개발/퍼블리싱 웹화면을 **태그 없이** 측정(북마크 한 번)해서
- **이름표 없이 자동 짝맞춤 → river 확인 → 값 차이 리포트**. 외부 통신 없음.
- 자세한 사용법: [`plugin-value-qa/README.md`](plugin-value-qa/README.md)

## 폴더 안내

| 폴더/파일 | 내용 |
|---|---|
| **`plugin-value-qa/`** | ⭐ 현재 제품 (값 기반 플러그인 + 태그없는 수집기 + 북마크) |
| **`validation-auto/`** | 현재 방식(태그없는 자동 짝맞춤)의 **재현 가능한 자가검증** (`npm test`) |
| `validation/` | 초기 "태그 기반 값비교" 증명용 예시 (구버전, 보관) |
| `code.js` · `ui.html` · `manifest.json` (루트) | ⚠️ **구버전 플러그인** (이미지·OCR·픽셀diff 기반). 방치·대체됨. 검수결과서의 핀·이력 개념은 여기서 재활용 예정 |

## 검증
```
npm i && npm test
```
현재 값 트랙이 태그 없이 자동 짝맞춤(17/17)하고 심어둔 결함을 검출하는지 실제 플러그인으로 재현합니다.

## 방향 (요약)
- **웹/모바일웹** = 값 대조 (지금 것) · **앱/모바일앱** = 화면 캡처 → AI 비전 대조 (예정).
- 최종 산출물 = **개발화면 검수결과서** (디자인+개발 나란히·빨간 체크·이력, Figma & HTML).
