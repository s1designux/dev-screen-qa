# 검증 세트 — 새 검수 방식이 되는지 증명하기

이 폴더는 **"피그마 정답값을 그 자리에서 읽어 개발화면과 대조한다"**는 새 방식이
실제로 결함을 잡아내는지 증명하기 위한 예시 묶음입니다. 외부 통신·비용 없음.

## 무엇을 증명했나 (결과)

일부러 **10군데 틀리게** 만든 개발화면을 대조한 결과, **10개 결함을 전부 자동 검출**했습니다.

| 검사 종류 | 잡아낸 것 | 신뢰도 |
|---|---|---|
| 속성 (색·글자크기·굵기·모서리·여백) | 6개 결함 | ⭐ 헛경보 0 — 완벽 |
| 크기 (너비·높이) | 검색창·아바타 | ⭐ 정확 |
| 위치 (가로) | 버튼 40px 밀림 | ⭐ 정확 |
| 존재 (요소 추가/누락) | 없던 배너 | ⭐ 정확 |
| 위치 (세로) | — | ⚠️ 구조가 바뀌면 아래가 줄줄이 밀림 → **원인 밑으로 묶어** 처리 |

> **핵심 발견:** "무엇이 어떻게 생겼나(색·크기·글자)"는 완벽하게 잡습니다.
> "세로 위치"는 요소 하나가 추가/삭제되면 그 아래가 전부 밀려서, 이 밀림은
> 개별 오류로 세지 않고 **원인(추가된 요소) 밑으로 묶어** 보여줍니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| **report.html** | 📊 비교 리포트. **더블클릭해서 열면** 바로 결과가 뜹니다. |
| `measure-design-figma.json` | 정답값 — 실제 피그마 예시 대시보드에서 읽어온 스냅샷 |
| `measure-dev.json` | 개발 실측값 — 아래 dev.html을 collect.js로 잰 결과 |
| `dev.html` | 예시 개발화면 (일부러 10군데 틀림). 파일 안 주석에 결함 위치 표시 |
| `design.html` | 정답의 **설계도**. 이 값 그대로 피그마 예시 파일을 만들었습니다 |
| `collect.js` | 🔧 개발화면 측정 수집기 (브라우저 콘솔용) |

> 정답 예시 디자인은 피그마 파일에 있습니다: **"검수 검증 · 예시 대시보드(정답)"**
> (드래프트에 생성됨). report.html의 정답값은 이 피그마 파일에서 읽어온 것입니다.

## 직접 확인하는 법

**가장 빠른 길 — 그냥 결과만 보기**
1. `report.html`을 더블클릭해서 브라우저로 엽니다. → 결과가 바로 보입니다.

**개발화면을 직접 다시 재보기 (선택)**
1. `dev.html`을 브라우저로 엽니다.
2. `F12` → **Console** 탭에 `collect.js` 내용을 통째로 붙여넣고 Enter.
3. `measure-dev.json`이 자동 다운로드됩니다.
4. `report.html`을 열고 **②번 칸**에 방금 받은 파일을 넣으면 그 값으로 다시 비교합니다.

## 실제 1버전 흐름 (이 검증이 뒷받침하는 것)

```
💻 개발자: collect.js로 값 수집 → 측정 파일 전달
🎯 디자이너: 피그마에서 정답 프레임 선택 → 플러그인이 그 값을 라이브로 읽음
📊 플러그인: 정답 ↔ 개발 대조 → 결함·밀림 여파 정리해서 표시
🗂️ 이력 저장
```

이 검증 세트에서 "플러그인이 피그마를 라이브로 읽는" 부분은
`measure-design-figma.json`(피그마에서 실제로 읽어온 스냅샷)으로 대신했습니다.
플러그인이 완성되면 이 스냅샷 단계가 자동화됩니다.
