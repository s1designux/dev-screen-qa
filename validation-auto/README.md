# 자가검증 — 태그없는 자동 짝맞춤 (재현 가능)

지금 팔고 있는 **값(웹) 트랙의 핵심 두 가지** — 태그 없는 측정 + 이름표 없이 자동 짝맞춤 — 이
진짜 작동하는지 **저장소 안에서 재현**한다. (예전 `../validation/`은 태그 기반 옛 방식만 증명했음.)

## 무엇을 증명하나
실제 플러그인 `plugin-value-qa/ui.html`을 헤드리스 크롬으로 그대로 구동해서:
- 정답(피그마에서 읽은 값) + 개발(태그없이 측정한 값)을 넣고
- **이름표를 완전히 무시한 채** 자동 짝맞춤 → 값 대조까지 돌린 뒤
- 아래를 자동 검사(assert):
  1. **자동 짝맞춤 17/17** — 의도된 짝을 모두 정확히
  2. **심어둔 결함 6개 요소** 모두 "실제 문제"로 검출 (색·글자크기·굵기·모서리·여백·크기, 총 10군데)
  3. **구조 차이 1곳(추가된 배너)만** — 헛경보 없음

하나라도 어긋나면 `npm test`가 실패(exit 1)한다.

## 실행
```
npm i        # puppeteer-core 설치 (최초 1회)
npm test     # = node validation-auto/test.js
```
- Chrome이 필요해요. 경로가 다르면 `CHROME_PATH=/경로/크롬 npm test`.

## 파일
| 파일 | 내용 |
|---|---|
| `test.js` | 검증 하네스 (ui.html 구동 → 짝맞춤·대조·assert) |
| `design-figma.json` | 정답 스냅샷 — 예시 대시보드를 플러그인 리더로 읽은 값 |
| `measure-dev.json` | 개발 측정 — `../validation/dev.html`(일부러 10군데 틀림)을 태그없는 수집기로 잰 값 |

## 다시 만들려면 (스냅샷 갱신)
- **개발 측정:** `../validation/dev.html`을 브라우저로 열고 `plugin-value-qa/collect.js`(또는 북마클릿) 실행 → `measure-dev.json`.
- **정답 스냅샷:** 예시 대시보드 피그마 프레임을 플러그인 ①정답 읽기로 읽어 저장(현재는 세션에서 읽은 값 커밋).
