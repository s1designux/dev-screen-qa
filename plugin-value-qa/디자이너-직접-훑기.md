# 디자이너가 테스트 계정으로 직접 훑는 법 (개발자 협조 최소화)

개발자에게 화면마다 부탁하지 않고, **테스트 계정만 받으면 river님이 직접** 여러 화면·상태·팝업을 훑을 수 있어요. 콘솔 붙여넣기 대신 **북마크 한 번 클릭**으로요.

## 1단계: 북마크 만들기 (딱 한 번만 설정)

1. 크롬에서 북마크바를 켭니다 (`⌘⇧B`).
2. `collect-bookmarklet.txt` 파일을 열어 **내용 전체를 복사**합니다. (`javascript:`로 시작하는 긴 한 줄)
3. 북마크바에서 마우스 오른쪽 클릭 → **페이지 추가…(Add page…)**
4. **이름**: `📐 화면 측정` / **URL**: 복사한 내용 붙여넣기 → 저장.

> ⚠️ 회사 사이트가 보안정책(CSP)으로 북마클릿을 막으면, 대신 이 폴더의 `collect.js`를 콘솔에 붙여넣는 방식을 쓰면 돼요(같은 결과).

## 2단계: 훑기 (화면마다 클릭 한 번)

테스트 계정으로 로그인한 뒤, 검사할 화면마다:

1. 그 화면을 띄웁니다. (스크롤 긴 화면도 OK — **전체가 다 측정**돼요.)
2. **상태·팝업이 있으면 그걸 화면에 띄운 상태에서** — 팝업 열고, 버튼 상태 바꾸고.
3. 북마크 **`📐 화면 측정`** 클릭 → "측정 완료" 알림이 뜨고 파일이 다운로드됩니다.
   (파일명은 화면 제목으로 자동 구분돼요: `measure-<화면이름>.json`)
4. 다음 화면으로 이동해서 반복.

이렇게 **클릭만으로 여러 화면·상태를 쭉 훑어** 파일들을 모으면 돼요.

## 꼭 알아둘 점 (정직하게)

- **스크롤 긴 화면**: 자동으로 전체가 측정돼요. 걱정 안 해도 됨.
- **상태별 버튼·팝업**: 도구가 "안 눌린 팝업"은 못 봐요. river님이 **그 상태를 화면에 띄운 다음** 클릭해야 잡혀요. (검수하려면 어차피 그 상태를 봐야 하니 자연스러워요.)
- **측정 폭**: 창 폭에 따라 위치가 달라져요. 피그마 정답과 **같은 폭**(예: 1440·1920)에서 훑는 게 좋아요.
- 이 파일엔 색·크기·위치 **숫자값만** 들어요. 소스·개인정보·캡처 아님.

## 그다음
모은 `measure-*.json`들을 플러그인 ②번 칸에 하나씩 넣어 각 화면을 검수하면 돼요.
(화면 이미지도 나란히 보고 싶으면 → 각 화면에서 `⌘⇧P → Capture full size screenshot`으로 PNG도 받아 🖥 칸에 넣기)
