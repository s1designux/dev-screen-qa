# 값 검수 플러그인 (v1)

피그마 정답 값을 **그 자리에서 라이브로 읽어** 개발화면 실측과 대조하는 Figma 플러그인.
사진(픽셀) 비교가 아니라 실제 값·좌표·색·글자를 비교합니다.
**이름표(태그) 없이 자동으로 짝을 맞추고, river님이 확인**합니다. **외부 통신 없음.**

## 피그마에 올리는 법 (한 번만)
1. Figma **데스크톱 앱**을 엽니다.
2. 메뉴 → **Plugins → Development → Import plugin from manifest…**
3. 이 폴더의 **manifest.json** 을 선택합니다.
4. **Plugins → Development → 값 검수 (디자인↔개발 대조)** 로 실행됩니다.

## 쓰는 법 (개발자 협조 = 딱 1번 붙여넣기)
1. **개발자에게 부탁**: 개발화면을 브라우저로 열고 → F12 → Console → 이 폴더의
   **`collect.js`** 내용을 통째로 붙여넣고 Enter. → `measure-dev.json`이 다운로드됩니다.
   (**태그·소스 공유 불필요.** 콘솔에 딱 한 번 붙여넣기만.)
2. **① 정답 프레임** — 피그마에서 화면 전체 프레임을 선택하고 **[정답 값 읽기]**.
3. **② 개발 측정 파일** — 받은 `measure-dev.json`을 끌어다 놓기.
4. **③ 짝 확인** — 플러그인이 정답↔개발 요소를 **자동으로 짝지어** 보여줍니다(확신도 %).
   틀린 짝만 체크 해제하거나, 못 찾은 건 직접 골라주세요.
5. **[값 대조하기]** → 대조 리포트가 나옵니다.

### 🖥 개발화면 이미지 나란히 보기 (선택, 권장)
자동 짝맞춤이 흔들리는 복잡한 화면은 **눈으로 비교**가 제일 확실해요.
1. 개발자가 `collect.js` 실행한 콘솔에서 **⌘⇧P → `Capture full size screenshot`** → 화면 전체 PNG 다운로드.
2. 플러그인 **🖥 개발화면 이미지** 칸에 그 PNG를 끌어다 놓기.
3. → 피그마 **정답 프레임 오른쪽에 같은 너비로 개발화면이 놓여요.** 값 리포트(패널) + 두 화면 나란히(캔버스)로 검수.
   (먼저 ① 정답 값 읽기를 해야 어느 프레임 옆에 놓을지 알아요.)

## 리포트가 보여주는 것
- 🔴 **실제 문제** — 색·글자·크기·모서리·여백이 어긋난 곳 (**값·크기 기준**)
- 🧩 **구조 차이** — 디자인에 없던 요소 추가 / 정답에 있는데 개발에서 못 찾음
- ⬇️ **밀림 여파** — 구조가 바뀌어 아래로 밀린 요소는 원인 밑으로 묶음(개별 오류로 안 셈)
- 🟡 주의(미세) · ✅ 완전 일치 · ⚠️ 측정 폭이 다르면 경고
- 📄 **HTML 검수결과서 내보내기** — 이슈 목록(정답↔개발 값 + '열림' 상태)을 공유·인쇄용 HTML 1파일로

> **위치(x·y)는 '참고'로만** 표시하고 판정엔 안 넣어요 — 디자인 프레임엔 브라우저 목업·다른 페이지높이 때문에 절대 위치가 어긋나기 쉬워서요. 위치·레이아웃은 **개발화면 이미지 나란히(시각 검수)**로 봅니다.

## 정직하게 — 한계 & 팁
- 자동 짝맞춤은 완벽하지 않아요. 그래서 **③ 확인 단계가 반드시** 있습니다(확신도 낮은 짝=노란색부터).
- **팁: 정답은 '순수 화면' 프레임을 고르세요.** 브라우저 목업(가짜 주소창·탭)이 포함된 프레임을 고르면 목업 요소가 짝맞춤 노이즈를 만들어요.
- 작은 **아이콘(벡터 조각)은 자동 제외**해요(개발화면은 아이콘을 이미지 1개로 그려 값 비교 불가).
- **위치·레이아웃 정밀 검수는 값 대조로 안 하고** 개발화면 이미지 나란히(시각)로 합니다.

## 검증 (재현 가능)
`../validation-auto/`에 **커밋된 자가검증**이 있어요. 실제 플러그인 `ui.html`을 헤드리스 크롬으로 구동해:
- 이름표 완전히 무시하고 **자동 짝맞춤 17/17 정확**
- 심어둔 결함 요소 **6곳(색·글자크기·굵기·모서리·여백·크기 — 총 10군데)** 모두 검출
- **구조 차이 1곳(추가된 배너)만** — 헛경보 없음

실행: 저장소 루트에서 `npm i && npm test`. 자세한 건 `../validation-auto/README.md`.
> 참고: `../validation/`은 **옛 태그(data-qa) 기반** 값비교 예시로, 지금의 자동 짝맞춤은 다루지 않음(구버전 예시).
