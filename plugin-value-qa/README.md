# 값 검수 플러그인 (v1)

피그마 정답 값을 **그 자리에서 라이브로 읽어** 개발화면 실측과 대조하는 Figma 플러그인.
사진(픽셀) 비교가 아니라 실제 값·좌표·색·글자를 비교합니다. **외부 통신 없음.**

## 피그마에 올리는 법 (한 번만)
1. Figma **데스크톱 앱**을 엽니다.
2. 메뉴 → **Plugins → Development → Import plugin from manifest…**
3. 이 폴더의 **manifest.json** 을 선택합니다.
4. **Plugins → Development → 값 검수 (디자인↔개발 대조)** 로 실행됩니다.

## 쓰는 법
1. **레이어 이름 맞추기** — 정답 프레임 안의 각 요소 레이어 이름을
   개발쪽 이름표(`data-qa`)와 **똑같이** 지어둡니다. 예: `header/logo`, `stat/card-2`.
   (콘텐츠 샘플 영역은 `content/` 로 시작하면 위치·크기만 검사합니다.)
2. **① 정답 프레임** — 피그마에서 화면 전체 프레임을 선택하고 **[정답 값 읽기]**.
3. **② 개발 측정 파일** — 개발자가 `collect.js`로 뽑은 `measure-dev.json`을 끌어다 놓기.
4. 두 값이 준비되면 **대조 리포트**가 바로 뜹니다.

## 리포트가 보여주는 것
- 🔴 **실제 문제** — 색·글자·크기·모서리·여백이 어긋난 곳
- 🧩 **구조 차이** — 디자인에 없던 요소가 추가되거나, 있어야 할 게 빠진 곳
- ⬇️ **밀림 여파** — 구조가 바뀌어 아래로 밀린 요소는 원인 밑으로 묶어서 표시(개별 오류로 세지 않음)
- 🟡 주의(미세) · ✅ 완전 일치
- ⚠️ 정답과 개발의 **측정 폭이 다르면 경고**

## 아직 안 한 것 (다음 단계 후보)
- 이력 저장 / 여러 화면 관리 (기존 `개발화면 검수` 플러그인의 관리 골격과 합치기)
- 불일치를 캔버스에 핀으로 표시
- 세로 위치 밀림을 더 똑똑하게(구조 정렬 기반) 처리

## 검증
이 플러그인의 대조 로직은 `../validation/` 예시로 증명됐습니다 —
일부러 심은 결함 10개를 전부 자동 검출. 자세한 건 `../validation/README.md` 참고.
